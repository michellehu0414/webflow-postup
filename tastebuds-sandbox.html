Guerilla testing

I prototoyped my sketches and conducted guerilla tests with 5 participants.

Outcomes

Diners highly value deals.

Map is for location based search.

3 diners couldn't detect the map.

4 diners did not use the slider.

Recommendations

Given my findings, here are a few improvements and changes I planned to integrate into my low-fidelity wireframes:

Route 1

  • Make "strengths and weaknesses" selection values mutually exclusive
    (i.e. if “authentic” as a strength is selected, it can't be selected as a weakness)
  • Implement drag interaction design for star rating component
  • Clarify rules of rating system

Route 2

  • Add a 'map' button at the bottom of search results when in list view
  • Add 'sort by distance' into search filters
  • Add restaurant 'deals' into search results
